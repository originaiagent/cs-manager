# API契約ルール

## エンドポイント

- エンドポイントの追加はOK
- 既存エンドポイントの削除・パス変更は禁止

## レスポンス

- レスポンスのフィールド追加はOK
- 既存フィールドの削除・型変更は禁止（後方互換性維持）
- エラーレスポンスは `{ error: string }` 形式で統一

## 認証

- 認証が必要なエンドポイントは `Authorization: Bearer {token}` ヘッダー

## 変更後

- 変更後にAGENTS.mdの連携情報を更新すること
