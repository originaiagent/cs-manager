### Lステップ脱却・LINE移行（トム決定 2026-07-22）
🎯 最終ゴール: Lステップを解約し、LINE公式アカウント標準機能 + 自社ツール（cs-manager 等）でフォローする

背景 / 制約:
- LINE公式アカウントの Webhook は1アカウントに1つしか設定できず、Lステップが使用中。素直に切り替えるとLステップの受信起点機能（自動応答・メッセージ起点のタグ付け等）が止まるため移行タイミングが難しい、というのが出発点
- 併用期間は **Lステップの「LINE Webhook転送」オプション**（全プラン可・月5,500円税込・LINE素のwebhookデータをそのまま外部転送。Lステップ固有のタグ情報は含まれない）で作る方針をトムが承認済み
- cs-manager 側の LINE 受信・返信送信は実装済み（inbound 直受け + 送信アダプタ + cron）。未接続なのは go-live 設定（キー投入・config.scope_key・Webhook URL）のみ

フェーズ分解:
1. **L1 併用開始**: トムが Webhook転送オプションを申込み（課金発生・トム作業）→ cs-manager 受け口の転送対応（転送時に X-Line-Signature 等の認証ヘッダが付くかを実測し、必要なら転送用認証へ調整）→ LINE go-live（受信実証まで）。この段階では cs-manager は閲覧+下書きのみ、返信は従来どおりLステップ（二重返信防止）
2. **L2 CS対応の一本化**: 返信を cs-manager の承認制送信へ切替。Lステップは配信専用に格下げ
3. **L3 Lステップ利用機能の棚卸し**: 実際に使っている機能をトムにヒアリング（ステップ配信 / セグメント配信 / タグ管理 / リッチメニュー / 回答フォーム / 流入経路分析 等）→「LINE公式標準機能で足りる / 自社ツールで実装する / 使うのをやめる」の3分類マッピングを作りトム承認
4. **L4 代替実装**: L3 で「自社ツールで実装する」に分類された機能を構築（着手時に個別 goal 化。候補: セグメント配信・ステップ配信の自社ツール化）
5. **L5 切替・解約**: LINE Developers の Webhook を cs-manager 直結へ切替（即時反映・即時に戻せる）→ 本番動作確認 → **Lステップ解約**（Webhook転送オプションの課金も終了）

備考:
- Webhook を切り替えても Lステップの一斉配信・ステップ配信（こちらから送る push 系）は動き続ける。止まるのは受信起点の機能のみ。このため L4 完了前でも受信系だけ先に直結する選択肢がある
- 各フェーズの着手時期・優先順位はトム判断待ち（business_priority 未確定）。L1 の申込み（課金）と L3 の棚卸しヒアリングはトムの作業が必要
